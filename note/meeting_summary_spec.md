# 功能規格書：Gemini AI 文字分析與結構化摘要生成

## 1. 功能概述
本模組負責承接 Whisper API 辨識產出的會議逐字稿文本（.txt），透過呼叫 Gemini AI 進行內容提煉，並強制輸出結構化的數據格式，直接用於發送 Email 與歷史紀錄調閱。

## 2. Gemini AI 提示詞與輸出限制 (Prompt Design)
呼叫 Gemini API 時，必須注入以下系統指令（System Instruction），以確保輸出格式絕對穩定：

> **AI 角色定義：**
> 你是一位高效、精準的企業會議助理。請針對提供的會議逐字稿進行分析，並嚴格遵循結構化格式進行摘要輸出。
> 
> **輸出格式要求：**
> 必須包含且僅包含以下四個項目，請以清晰的條列式呈現：
> 1. **會議大綱**：精煉概括本次會議討論的核心主題。
> 2. **項目進度**：摘錄目前已完成、進行中或落後的專案執行進度。
> 3. **與會人待辦事項與期限**：列出每位與會人員被指派的具體工作、責任人姓名，以及明確的完成期限 (Deadline)。
> 4. **下次會議目標**：根據本次會議結論，定義下一次會議的預計討論重點。

## 3. 發起人確認與半小時自動發信邏輯
摘要生成後，後端需啟動定時任務（排程）追蹤狀態：

[ Gemini 生成摘要 ] 
           |
           v
   [ 寫入資料庫, 狀態=未確認 ] 
                   |
          +--------+--------+
          | (半小時內)       | (超過半小時未確認)
          v                 v
    [ 發起人手動點擊確認 ]   [ 系統自動判定 ]
            |                 |
            +--------+--------+
                     |
                     v
        [ 觸發 Gmail API 發送 Email ]

* **流程 A（手動確認）**：發起人在系統介面確認摘要無誤，點擊「確認發送」，狀態轉為 `TRUE`，立即發送 Email 通知所有與會人。
* **流程 B（逾時自動處理）**：若摘要寫入資料庫後 **30 分鐘內**，發起人未進行任何操作，後端排程系統將自動觸發發信機制，將此摘要寄出給所有與會人。
